## 一、端口隔离
### 1.工作原理
![](DICM/Pasted%20image%2020251127142117.png)

将相同 VLAN 对应的接口划分到一个端口隔离组，那么可以实现广播域变小。甚至一个接口可以属于一个广播域。<font color="#ff0000">一般公共资源</font>（打印机和服务器不需要划分到该端口隔离组）

#### 配置
##### 1.端口隔离
1. 首先划好 vlan
2. <font color="#ff0000">每个想要隔离的端口下都需要配置</font>
	```
 	[Huawei]interface GigabitEthernet 0/0/1
 	[Huawei-GigabitEthernet0/0/1]port-isolate enable group 1
 	[Huawei]port-isolate mode [all（二层、三层都隔离）/l2（只二层隔离，默认是这个）]
 	```
3. 此时就已经开启端口隔离，被隔离的设备之间无法互访
##### 2.开启 ARP 的 VLAN 内代理
<font color="#ff0000">如果非要端口隔离的两个设备通信呢？</font>
SW1：
```
interface Vlanif10
 ip address 192.168.1.254 255.255.255.0
 arp-proxy inner-sub-vlan-proxy enable
```
![](DICM/Pasted%20image%2020251127151501.png)


##### 注意：VLAN 内的 ARP 代理只针对端口隔离场景
只有 ARP 是从端口隔离下的设备发出的，ARP 代理才会理你，否则不理你
#### 隔离模式
##### 双向隔离
###### 1.L2隔离
如果没有做端口隔离，PC1访问 PC2属于 L2转发（交换机的角度），如果做了端口隔离，那么同一个 VLAN 同网段主机无法互访
###### 2. L2和 L3都隔离
如果做了端口隔离，并做了 VLAN 内的 ARP 代理，那么同一个 VLAN 下的同网段的主机互访，交换机是 L3转发，如果做了端口隔离，那么同一个 VLAN 同网段主机默认可以互访

##### 单向隔离
`am isolate GigabitEthernet0/0/2`：在端口1下配置，代表的意思是当前端口无法访问端口2（<font color="#ff0000">但是端口2可以进来</font>）

## 二、端口安全
### 产生背景-MAC 地址的攻击
<font color="#ff0000">攻击原理</font>：攻击者发出大量的源不断变化的数据帧，消耗完交换机的 MAC 地址表资源。导致后续交换机收到的所有的数据包都认为是未知单播包，并泛洪处理，消耗链路带宽和设备性能

### <font color="#ff0000">如何解决？</font>
#### 1.<font color="#7030a0"> 限制接口学习 MAC 地址的数量</font>：（配置静态 MAC 地址表）
- 原理：接口如果学习到的 MAC <font color="#ff0000">超过</font>设置的接口学习 MAC 地址的<font color="#ff0000">最大值</font>，该接口就不会再学习心新的 MAC地址，必须等待老化以后才可以。避免 MAC 地址表被消耗完 
- 配置命令：`[Huawei-GigabitEthernet0/0/1]mac-limit maximum 3`	![](DICM/Pasted%20image%2020251127160733.png)


#### 2. 交换机接口不学习 MAC 地址的场景
##### 1. 静态配置 MAC 地址绑定接口
```
[Huawei]mac-address static [MAC] interface [接口] vlan [ID]
[Huawei-GigabitEthernet0/0/1]mac-address learning disable
```
![](DICM/Pasted%20image%2020251128142859.png)

##### 2.三层互联场景
- 使用VLANif作为三层逻辑接口互联
- 通过ARP表实现通信，无需二层MAC地址表
![](DICM/Pasted%20image%2020251128144333.png)
对于 LSWS1：
```
interface GigabitEthernet0/0/3
 mac-address learning disable
 port link-type access
 port default vlan 1000
interface Vlanif1000
 ip address 100.1.1.1 255.255.255.0
```
对于 AR2：
```
interface GigabitEthernet0/0/0
 ip address 100.1.1.2 255.255.255.0 
```
此时 `LSW1` 的3号端口不会学习到 MAC 地址表，但是可以在 ARP 表学习到 MAC
##### 为什么会出现MAC 表没记录，但 ARP 表有记录？
- **MAC 地址表**：是**交换机二层转发**用的，`mac-address learning disable` 会阻止端口学习 MAC，所以 MAC 表中不会有 GE0/0/3 端口对应的 MAC
- **ARP 表**：是**三层接口通信**用的（记录 “哪个 IP 对应哪个 MAC”），它的生成和 “交换机端口是否学习 MAC” 无关，只要三层接口能收到 ARP 响应，就会记录 ARP 表项
- “端口不学习 MAC（影响 MAC 表）” 和 “三层接口记录 ARP（影响 ARP 表）” 是两个独立的机制

####  3.<font color="#7030a0">端口安全</font>
##### 1. 安全静态 MAC 地址
管理员<font color="#ff0000">手工配置</font> MAC 地址与接口的绑定关系，**可以限制接口下合法的 MAC 数量（默认是一个）**，<font color="#ff0000">安全接口 MAC 地址不会老化，也不会因为接口 DOWN 或者设备重启而消失</font>
- 优先是安全性最高，缺点是繁琐。
- 一般会配置连接服务器的接口（服务器一般不会更改接口）。
- 适用于小规模网络
##### 2.安全动态 MAC 地址
交换机自动学习，将学习到的 MAC 转换成安全的 MAC ,<font color="#7030a0"> 安全接口MAC 地址不会老化</font>，<font color="#ff0000">但是会因为接口 DOWN 或者设备重启而消失</font>
- 相比安全静态 MAC 安全性会差一些
- 一般适用于访客网络
- 需要设置老化时间
![](DICM/Pasted%20image%2020251127164406.png)
##### 3. 安全 stickyMAC 地址
交换机自动学习，将学习到的 MAC 转换成安全的 MAC ，并且<font color="#ff0000"> 安全接口MAC 地址不会老化，也不会因为接口 DOWN 或者设备重启而消失</font>
- 相比安全静态 MAC 安全性会差一些
- 配置简单（打开端口安全并设置为 stickyMAC 模式，后续没有配置）
- 适用于大型网络
![](DICM/Pasted%20image%2020251127164445.png)
##### 端口安全的保护行为
1. shutdown 接口
2. 丢弃数据
3. 丢弃数据并报警（默认）

## 三、DHCP Snooping--DHCP 的窥探
### 1.工作原理
交换机开启了 DHCP Snooping 后，会监听交换机的 DHCP 报文，然后根据 DHCP 的 <font color="#ff0000">ACK</font>报文，形成 DHCP Snooping 表项。
	![](DICM/Pasted%20image%2020251128151250.png)
### 2. DHCP Snooping 的配置
```
[Huawei]dhcp snooping enable  //全局下配置
 [Huawei-vlan1]dhcp snooping enable  //VLAN下配置
 
 [Huawei]dis dhcp snooping user-bind all   //查看DHCP Snooping 的命令
```

### 3. DHCP Snooping 的表项
```
[Huawei]dis dhcp snooping user-bind all
DHCP Dynamic Bind-table:
Flags:O - outer vlan ,I - inner vlan ,P - map vlan 
IP Address       MAC Address     VSI/VLAN(O/I/P) Interface      Lease           

--------------------------------------------------------------------------------

192.168.1.252    5489-9809-1569  1   /--  /--    GE0/0/2        2025.11.29-15:04
--------------------------------------------------------------------------------
print count:           2          total count:           2  
```
### 4. 关于 DHCP 的攻击防范
1. DOS 攻击（拒绝服务攻击）
	- 攻击原理：
		1. <font color="#ff0000"> 仅篡改 DHCP 报文中的 CHADDR 字段</font>：攻击者发送大量 DHCP 报文的 `CHADDR` 伪造成随机 MAC，大量申请 IP 地址，可以很快消耗 DHCP 地址池中的地址
		2. <font color="#ff0000">同时篡改 CHADDR 和数据帧的源 MAC</font>：攻击者发送大量DHCP 报文的 `CHADDR` 伪造成随机 MAC，**数据帧的源 MAC 也伪造成和 CHADDR 一致**，大量申请 IP 地址，可以很快消耗 DHCP 地址池中的地址
	- 预防措施
		- 攻击1：在交换机上开启 DHCP 报文的检查（检查 CHADDR 和源 MAC 是否一样，如果不一致就丢弃）
		- 攻击2：在交换机的接口下配置用户绑定表的数量
			- 注意：user 绑定表有两种方式：1. DHCP Snooping 生成；2. 手工配置（用于固定 IP 的设备）
2. DHCP 服务器的防冒攻击
	- 攻击原理：攻击者**仿冒DHCP服务器**或者**误接 DHCP 服务器**去分配错误的IP地址，导致客户端获取地址后无法访问相应资源。
	- 防范措施：交换机开启 DHCP Snooping 并且朝着 DHCP 服务器的接口配置为 DHCP Snooping 的可信端口
	- 配置命令：
		1. 全局下配置
			`[Huawei]dhcp snooping enable`
		2. VLAN 下配置
			`[Huawei-vlan1]dhcp snooping enable`
		3. 配置 DHCP 可信端口

## 四、DAI--动态 ARP 检测

### 中间人攻击
1. 攻击原理：攻击者发出免费 ARP 报文，ARP 载荷中的 <font color="#ff0000">SenderIP 填写成被攻击者的 IP</font>，但是 <font color="#ff0000">senderMAC 填写成攻击者自己的 MAC</font>
2. 预防措施：
	- 开启 DAI 功能，开启 DAI 的前提是必须有 user-bind 表（可以是静态的手工配置或者动态 DHCP Snooping 生成）
	- <font color="#ff0000">如果收到的 ARP 报文的载荷的 senderIP 和 senderMAC 和 user-bind 表中的不一致，那么就丢弃该 ARP 报文</font>。
	`[Huawei-vlan1]arp anti-attack check user-bind enable`
	![](DICM/Pasted%20image%2020251128173820.png)

3. 配置
	```
	vlan1
	[Huawei-vlan1]arp anti-attack check user-bind enable
	```

### 1.端口安全实验

	interface GigabitEthernet0/0/1
	 mac-limit maximum 3
	```
2. 此时，给 PC1改 MAC 地址来模拟交换机端口对 MAC 地址的学习
	- 从 `54-89-98-C3-21-11`、`54-89-98-C3-21-22`、`54-89-98-C3-21-33` 和 `54-89-98-C3-21-44`
3. 当数量超过三个的时候（即到了 `54-89-98-C3-21-44`），交换机会报警告 `MAC address learning reached the limit`，此时不会再学习1端口下新的 MAC 地址，但其他端口正常

### 2. DHCP Snooping 实验
#### 案例1：DHCP Snooping 的配置
DHCP Snooping 核心是通过 **DHCP 的 ACK** 报文来生成并维护 `DHCP Snooping 绑定表`
![](DICM/Pasted%20image%2020251128154317.png)
1. 首先在 vlan 下配置 DHCP 地址池
	```
	interface Vlanif1
	 ip address 192.168.1.254 255.255.255.0
	 dhcp select interface
	```
2. 在全局开启DHCP Snooping，再去目标 vlan 端口下配置DHCP Snooping
	```
	[Huawei]dhcp snooping enable  //全局下配置
	 [Huawei-vlan1]dhcp snooping enable  //VLAN下配置
	```
	注意：<font color="#ff0000">在不配置 vlan 的情况下，交换机的所有接口均默认在 vlan1下</font>，因此只需要配置 vlanif1的地址池、vlan1的 dhcp snooping enable
3. 此时，PC1和 PC2获取 dhcp 地址后，在发送 ACK 报文，就会被交换机记录下绑定表，用 `[Huawei]dis dhcp snooping user-bind all` 查看表内容

#### 案例2：“仅篡改 DHCP 报文中的 CHADDR 字段”的 DOS 攻击的防范
![](DICM/Pasted%20image%2020251128155725.png)
1. 首先，跟[案例1：DHCP Snooping 的配置](#案例1：DHCP%20Snooping%20的配置)一样的配置
2. 配置 DHCP 报文的检查：检查 CHADDR 和源 MAC 是否一样，如果不一致就丢弃
```
第一种配置方法：
[Huawei]vlan 1
[Huawei-vlan1]dhcp snooping check dhcp-chaddr enable 

第二种配置方式
[Huawei]dhcp snooping check dhcp-chaddr enable vlan 1

#### 案例3：“同时篡改 CHADDR 和数据帧的源 MAC” 的 DOS 攻击防范
![](DICM/Pasted%20image%2020251128160727.png)

1. 首先，跟[案例1：DHCP Snooping 的配置](#案例1：DHCP%20Snooping%20的配置)一样的配置
2. 在端口1下**配置接口用户绑定表的数量为3**（**即最多允许 3 个终端通过该端口获取合法 DHCP IP**）
```
interface GigabitEthernet0/0/1
 dhcp snooping max-user-number 3
```
如图所示：<font color="#ff0000">当端口下用户绑定表的 MAC 地址到达3个后，后续将不会绑定新的 MAC 地址</font>，超过的终端连 DHCP 请求都会被丢弃

此时在端口下配置 `dhcp snooping trust` 后，<font color="#ff0000">标记为信任端口，允许DHCP服务器的响应报文通过</font>

此时交换机的 `DHCP Snooping 绑定表` 也学习到了相关 MAC

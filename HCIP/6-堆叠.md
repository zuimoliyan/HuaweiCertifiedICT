# 一、堆叠产生背景

部署交换网络容易出现单点故障，然后业务中断

# 二、堆叠的定义
1. 堆叠的线缆将 `1~16` 个交换机连接起来成为一个逻辑的交换机。
	华为早期使用专门的堆叠模块，模块是连接在背板上的
	后期华为设备也是使用业务接口，1个或多个划分到堆叠口中，堆叠口必须是<font color="#ff0000">10G 或以上</font>。
2. 堆叠和 LAG 技术是一堆黄金组合，如果使用堆叠不使用 LAG 的话，那么就没有发挥堆叠的功能

# 三、堆叠的好处和相关角色
## 1、好处
1. 运维简化
	堆叠后多台物理机合并为单台逻辑机，全网仅呈现一个管理节点、一套配置体系，彻底消除传统多交换机独立组网的运维繁琐问题
2. 高可靠
	堆叠实现**设备不坏、链路不断、业务不停**，是园区组网中保障网络稳定性的核心技术之一
3. 无环
	堆叠内的物理交换机，通过**专用堆叠协议**实现内部互联，所有堆叠成员在逻辑上是**一台设备**，堆叠链路属于设备**内部转发通道**，而非二层网络的互联链路，因此**不会产生二层环路**
4. ECMP--负载分担
	堆叠将多台物理交换机的转发能力、端口带宽**整合为一个整体**，可基于**等价多路径（ECMP）** 实现流量的负载分担，同时结合链路聚合，把多条物理链路虚拟为一条高带宽逻辑链路

## 2、堆叠设备的角色
- Master：堆叠内**仅有 1 台**Master，是整个堆叠系统的核心管控单元，全权承担堆叠的**管理平面**与**控制平面**所有工作

- Standby：堆叠内**仅有 1 台**Standby，作为 Master 的专属热备节点，全程处于热备就绪状态，不参与堆叠的管理与控制工作，仅完成 Master 核心数据的实时同步，在 Master 故障时无缝接管所有工作，保障堆叠系统无中断运行

- Slave：堆叠内可存在**多台**Slave 设备，属于堆叠的纯业务节点，无管理与控制平面功能，仅接收 Master 下发的配置与核心数据

## 3.1、堆叠的优先级竞争
堆叠的 Master 选举**默认是非抢占模式**：即使后续有「优先级更高」的设备加入堆叠，**已当选的 Master 也不会被替换**，新设备自动成为 Slave
1. 手动配置的堆叠优先级（权重）
	- 设备可配置堆叠优先级，取值范围一般为`1~255`（**数值越大，优先级越高**）；
2. 设备的运行时长（上线时间）
	- 若所有设备的**堆叠优先级完全相同**，则比对**设备已启动并运行的时长**
	- 该规则的目的：优先保留已稳定运行、承载业务的设备为主，减少堆叠角色切换对业务的影响
3. 设备的 MAC 地址
	- **MAC 地址更小**的设备，自动当选 Master
## 3.2、堆叠分裂 / 原 Master 故障
1. 原 Master 设备故障宕机 / 被移除
	堆叠内所有 Slave 设备，**立即触发重新选举**，依旧严格遵循「优先级→运行时长→MAC 地址」规则，从剩余 Slave 中选出新 Master，堆叠业务不中断
2. 堆叠链路断开，堆叠分裂成多个分片
	1. 原堆叠分裂为 2 个（或多个）独立分片，**每个分片都会认为「原 Master 丢失」**
	2. 每个分片内部，都会按选举规则**各自选出 1 台本地 Master** → 全网出现**多台 Master（双主 / 多主）**
	3. 多台 Master 拥有相同的堆叠 IP、MAC，会引发全网 IP/MAC 冲突、广播风暴 → **触发 DAD 双主检测**，按 DAD 规则仲裁后阻塞故障分片的业务端口

# 四、术语

1. 域 `domainID <1~65535>`：描述堆叠域，数字描述的，如果两个或者多个设备要堆叠成为一个设备，那么 `domainID` 要<font color="#ff0000">一致</font>
2. 优先级 `priority <1~255>`：因为堆叠的成员设备需要选择主备，<font color="#ff0000">优先级默认是100，选大的</font>
3. 成员 ID：用来标识和管理交换机的，<font color="#ff0000">ID唯一</font>
4. 堆叠口：每台设备默认两个堆叠口
	![](DICM/Pasted%20image%2020251227144455.png)
5. 物理堆叠口：划分到堆叠口的物理接口，用于交换机的连接形成堆叠系统

## 1、环式堆叠
![](DICM/Pasted%20image%2020251227145852.png)
将 `SW3、SW4、SW5、SW6` 堆叠，但是如何破除环路？
从主设备<font color="#ff0000">成员 ID</font>较大的堆叠口开始数起，数(n/2)+1根线，这根线就是破环线，它两端的端口都在阻塞状态
- <font color="#ff0000">（n/2）+1取整数</font>
- **数线方向是 “按堆叠 ID 从小到大的顺序遍历成员设备的堆叠端口”**
	![](DICM/Pasted%20image%2020251227151256.png)
	假设 SW3的 ID 是3、SW4的 ID 是4、SW5的 ID 是5、SW6的 ID 是6，也就是说 master 是 SW6
	此时从 stack2(假设设备的两个堆叠口都是0口和1口)开始往其他设备数：
	1. **LSW6↔LSW5**
	2. **LSW5↔LSW4**
	3. **LSW4↔LSW3**
	4. **LSW3↔LSW6**
	所以阻断 LSW4的2口和 LSW3的1口

# 六、DAD--双主检测
## 什么是 DAD？
**DAD（Duplicate Address Detection，重复地址检测）** 是三层网络中用于**检测接口配置的 IP 地址是否被同网段其他设备重复使用**的协议功能，核心目的是**规避 IP 地址冲突**，防止因地址重复导致的通信中断、数据转发异常问题

## DAD 核心原理
1. 设备在接口启用 DAD、配置 IP 后，主动向网段内发送探测报文，确认该 IP 是否已被其他设备占用，无冲突则正式启用该 IP，有冲突则禁用本接口 IP 并告警。

2. 堆叠口断开触发 DAD（双主检测）后，**被判定为 “备 / 失败方” 的整个堆叠分片上，除<font color="#ff0000">保留端口与堆叠成员端口</font>外的所有业务端口会被 Error - Down（阻塞）**
	- 也就是说备用设备除了：堆叠口、DAD 口、管理口（MEth 管理口）以外全部阻塞

### 完整处理流程（堆叠口断→DAD 触发）
1. 堆叠链路中断，系统分裂为多个独立分片，各分片均认为自身是主设备，出现双主冲突。
2. DAD 通过检测链路互发竞争报文，比较分片优先级。
3. 低优先级分片被惩罚，执行端口阻塞：除保留端口和堆叠成员端口外，所有业务端口 Error - Down。
4. 高优先级分片正常运行，保障核心业务不中断。
5. 修复堆叠链路后，分裂分片合并，<font color="#ff0000">被惩罚设备自动重启</font>，业务端口恢复正常。


## 1、双主（堆叠口或者链路故障）

1. 管理平面--METH 口 IP 地址会冲突
2. 控制平面--OSPF 或者 BGP 协议、链路聚合有问题
3. 数据转发--VLANIF   IP 地址冲突
## 2、DAD 的配置端口

1. 管理口：管理口（MEth0/0/0）是设备的带外管理接口，仅用于设备运维管理，DAD 配置以检测管理 IP 冲突为目标
2. 直连接口：先将物理口切为三层模式（`undo portswitch`），配置 IP 后启用 DAD
3. `eth-trunk`：DAD 配置在聚合逻辑口上（而非成员口），先创建三层 Eth-Trunk、配置 IP，再启用 DAD
直连接口和 `eth-trunk` 不能一起用，其他可以双保险


# 七、堆叠综合实验

## 1、直连接口配置 DAD
图片上的 DAD 接口是<font color="#ff0000">直连接口</font>： `GE0/0/24`
![](DICM/Pasted%20image%2020251229110508.png)

SW1：
配置Switch1的堆叠优先级为150，Domain ID为100
将Switch1的业务口GE0/0/11～GE0/0/12加入堆叠端口1/1
```
stack member 1 domain 100      #域100，成员编号1
stack member 1 priority 150    # 优先级150（主交换机）
#
int stack port 1/1             #成员1的1号堆叠逻辑端口
 port member-group interface GE0/0/11 to GE 0/0/12     # 绑定物理端口
#
int g 0/0/24
 dual-active detect mode direct   # 直连DAD检测
```

SW2：
配置 Switch2 的堆叠优先级为120，Domain ID 为100
将 Switch2 的业务口 GE0/0/11～GE0/0/12加入堆叠端口1/1
```
stack member 1 domain 100     #重启前临时ID=1，基于1配置
stack member 1 priority 120
stack member 1 renumber 10 inherit-config   #重启后ID=10，继承配置 
#
int stack port 1/1                          #成员2的1号堆叠逻辑端口
 port member-group interface GE0/0/11 to GE 0/0/12      # 绑定物理端口
#
int g 0/0/24
 dual-active detect mode direct   # 直连DAD检测
```


## 2、eth-trunk 口配置 DAD
![](DICM/Pasted%20image%2020251229110641.png)
SW1：
在堆叠系统上配置Eth-Trunk1接口代理方式双主检测功能，<font color="#ff0000">只需要在其中一台配置即可</font>
```
stack member 1 domain 100
stack member 1 priority 150
#
int stack port 1/1
 port member-group interface GE0/0/11 to GE 0/0/12 
#
int eth-trunk 1
 mode lacp-static
 trunkport g 0/0/8
 trunkport g 0/0/9
 
 dual-active detect mode relay
```

SW2：
```
stack member 1 domain 100
stack member 1 priority 120
stack member 1 renumber 10 inherit-config
#
int stack port 1/2
 port member-group interface GE0/0/11 to GE 0/0/12 
```

SW3：
在SwitchC上配置Eth-Trunk1的代理功能
```
int eth-trunk 1
 mode lacp-static
 trunkport g 0/0/8 to g0/0/9
 
 dual-active proxy  # 开启DAD代理（转发堆叠的DAD检测报文）
```
这里查看堆叠口：`dis eth 1`
	堆叠后发出的 System ID 是主设备的 `System ID + 1`，但是端口编号 ID 不一样

## 4、堆叠与 OSPF
在 SW 上运行 OSPF 和堆叠，在 AR 上运行 OSPF
![](DICM/Pasted%20image%2020251229111726.png)
此时如果堆叠线断了：
1. 两个设备会以相同的Router ID（1.1.1.1）建立OSPF邻居，导致LSA序列号持续递增
2. dis ip rou 1.1.1.1显示两条等价路由，分别指向不同下一跳（100.1.1.2-AR2的0口和200.1.1.2AR2的1口）
3. 自动Router ID调整：<font color="#ff0000">华为设备检测到冲突后会自动改用其他Router ID（如接口IP），避免持续震荡</font>

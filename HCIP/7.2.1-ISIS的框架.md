## 1、邻居表
### 1. ISIS 的 Hello 报文
#### 标准 P2P 网络和华为 P2P 网络邻居建立过程
<font color="#ff0000">首先在3way 情况下都是三次握手</font>
![](DICM/Pasted%20image%2020251219162438.png)
1. 首先都是 `Down` 的状态
2. AR1向 AR2发送 `Hello` 报文，当 AR2接收到之后，变为 `Init` 状态（因为发过来的报文中的 Nei 为空，即：<font color="#ff0000">AR1并没有收到 AR2的 System ID</font>）
3. AR2向 AR1发送 `Hello` 报文，包含自己的 `System ID` 和对端的 `System ID`
4. AR1收到后变为 UP 状态（因为其中包含自己的 System ID）
5. AR1向 AR2发送 Hello 报文，AR2变为 UP 状态

##### 1. 标准的 ISIS 建立邻居的过程
![](DICM/Pasted%20image%2020251222184311.png)
1. 三次握手后，互发 CNSP，假设此时 AR2上有 AR1没有的路由，此时 AR1会向 AR2发送 PSNP 请求
2. 此时 AR2收到后会发出 LSP，包含路由信息
3. AR1收到后回复 PSNP 确认信息
##### 2. 华为的 ISIS 建立邻居的过程
![](DICM/Pasted%20image%2020251222183504.png)
1. 互发 LSP（直接发出路由信息）
2. 互相接收到后都回复 PSNP 确认
3. 再互发 CSNP 用于**全局对账，保证 LSDB 一致**

##### 1. ISIS 的 broadcast 中 DIS 选举规则（优先级 + 系统 ID）

1. IS-IS 的 DIS 选举遵循**明确的优先级规则**，<font color="#ff0000">无抢占机制</font>，且同时存在**备份 DIS**（用于 DIS 故障时接替工作）
	- 每个 IS-IS 接口都有一个**DIS 优先级**，取值范围 `0~127`，华为设备**默认值为 64**。
	- 优先级**数值越大**，越优先成为 DIS；优先级为 0 的接口**仍可参与选举**（<font color="#ff0000">区别于 OSPF 中优先级 0 不能成为 DR</font>）

2. **无抢占机制**：即使后续接入优先级更高的设备，也不会替换当前的 DIS，直到当前 DIS 故障或接口 Down

3. **Level-1 和 Level-2 独立选举 DIS**

4. 选举时间: 固定20秒等待期，确保网络拓扑稳定
	- 与 OSPF 不同，OSPF 是如果区域中有 DR，则可以跳过等待时间
##### 2. DIS 的注意事项

PSNP 特性: 在 MA 网络中没有确认机制，依赖 DIS 周期性发送 CSNP 保证同步

## 2、LSDB
### 1. `Level-1` 的实节点 LSP
<font color="#ff0000">LSDB 的样式</font>
![](DICM/Pasted%20image%2020251223134028.png)
`0000.0000.0001.00-00*` 其中 `*` 表示这个 LSP 是自己的

- **LSPID**：System ID + 伪节点 ID + 分片 ID。<font color="#ff0000">如果级别确定了，那么 LSPID 就可以唯一的描述一条 LSP</font>
	1. System ID：用来标识 ISIS 节点的，类似于 OSPF 的 RID，长度为6字节
	2. 伪节点 ID
		- `00`：表示是一个实节点的 LSP
		- 非 `00`：表示的是一个伪节点 LSP
	3. 分片 ID：从 `00` 到 `FF`，一共可以携带256片 LSP，当 LSP 的大小超过 `1497`（默认），就会自动分一片 LSP 用来携带拓扑信息或者路由信息
- **Seq Num**
    LSP 序列号，是 32 位递增的整数。每次更新 LSP 时序列号 + 1，用于判断 LSP 的新旧（序列号大的 LSP 更新）。
- **Holdtime**
	LSP 剩余生存时间（单位：秒）。LSP 的最大生存时间默认 1200 秒，每经过一个周期（默认 30 分钟）会刷新并重置 Holdtime
	若 Holdtime 归 0，LSP 会被标记为过期
- **Length**
	LSP 的长度（单位：字节），表示该 LSP 实际携带的链路状态信息的大小
- **ATT/P/OL**
	1. **ATT**（Attachment）：<font color="#ff0000">仅 Level-1 LSP 有此标志</font>
		1. 值为 1 表示该设备是 Level-1-2 设备 (可连接到 Level-2 区域)
		2. 当值为  0 时：表示该 Level-1 设备只是纯 Level-1 设备（无法连接到 Level-2 区域）
	2. **P**（Pseudonode）：值为 1 表示该 LSP 是伪节点 LSP（由 DIS 生成）
	3. **OL**（Overload）：值为 1 表示该设备处于过载状态（不参与流量转发）
#### 1. 内容
描述的是 `Level-1` 的计算<font color="#ff0000">节点</font>的拓扑信息（TLV = 2 IS Neighbor TLV）和网络信息（ip reachability TLV TLV128）。类似于 OSPF 的 LSA1
#### 2. 泛洪范围
一个物理区域内泛洪，如例题是在 `49.0000.0000.000X.00`，所以是在 `49` 区域泛洪

#### 3. 谁产生
Level-1中的所有路由器都会产生

### 2. `Level-1` 的伪节点 LSP

#### 1. 内容
描述的是 `Level-1` 的计算<font color="#ff0000">伪节点</font>的拓扑信息和网络信息（没有网络信息），类似于 OSPF 的 LSA
#### 2. 泛洪范围
一个物理区域内泛洪，如例题是在 `49.0000.0000.000X.00`，所以是在 `49` 区域泛洪

#### 3. 谁产生：DIS 产生
1. DIS 的特点
	- DIS 没有备份
	- DIS 发送 Hello 报文默认是3秒，老化时间是10秒
	- DIS 优先级取值范围是 `0~127`，0也可以竞选 DIS
2. DIS 的选举
	- 优先级数值越大越优先；优先级相同情况下，MAC 地址越大越优先

### 3. Level-2的实节点 LSP
#### 1. 内容
1. `Level-2` 区域的拓扑信息和网络信息，类似 OSPF 的 LSA1
2. level1区域计算出来的 level1路由信息，挂在 level2的实节点的 LSP 上，类似 OSPF 的 LSA3

#### 2. 泛洪范围
在逻辑的骨干区域泛洪
#### 3. 谁产生
`Level-2` 的路由器产生

### 4. Level-2的伪节点 LSP
#### 1. 内容
描述的是 `Level-2` 的伪节点的拓扑信息（没有网络信息），类似0SPF 的 LSA2
#### 2. 泛洪范围
在逻辑的骨干区域泛洪（在 level2的路由器所在区域泛洪）
#### 3. 谁产生
DIS 产生

## 3. 开销风格
配置命令：
```
[Huawei-isis-1]cost-style ?
  narrow             Set cost style to narrow
  wide               Set cost style to wide
```

### 1. narrow 模式（99%不用）
1. 接口开销最大可配置为63
2. 路由传递到其他区域的时候，端到端开销可能无法正确通过 `Level-2` 的 LSP 携带过去
3. narrow 不能携带 SUB 的 TLV，<font color="#ff0000">不能打 TAG</font>
	- 接口的开销最大的是6位，端到端开销是10位

### 2. wide 模式
配置：ISIS 下 `cost-style wide`
注意事项：
1. <font color="#ff0000">全网设备必须统一使用 wide 模式，否则会导致路由计算异常</font>
2.  当 wide 模式设备是 DIS 时，可以正常计算路由
	当 narrow 模式设备是 DIS 时，路由计算会失败
- **Wide 模式** 支持**完整的 24 位 System ID+6 位 SEL**（NSAP 地址），能正确传递、识别全网设备的真实路由标识；
- **Narrow 模式** 仅支持**6 字节固定 System ID**，且会强制对 LSP 的**ATT、OL、IS-Type** 等关键标识做**压缩编码**，导致非 Narrow 设备无法解析 LSP，全网链路状态数据库（LSDB）同步失败，最终路由计算瘫痪。

### 拓展
`auto cost enable`： ISIS 的**接口带宽自动折算开销**命令
	让设备**根据接口实际带宽，自动计算并赋值 ISIS 接口开销**，替代手动配置的 `isis cost` 固定值，实现**带宽与路由开销匹配**，让 ISIS 优选高带宽链路转发流量。
<font color="#ff0000">华为设备默认关闭</font>
公式：`(10*参考带宽)/物理带宽`

配置命令：
```
isis 1
 auto-cost enable
```

配置优先级： `接口手动配置 isis cost` > `auto-cost 自动折算` > `默认值10`
```
interface GigabitEthernet 0/0/3
 isis cost 20      // 手动配置开销，覆盖自动折算结果
```

### 1、路由优先级
`Level-1` > `Level-2` > `Leak(泄露的)`
`leak`：`Level-2` 计算的路由引入到 `Level-1`，携带 Up/Down 位
- 如果是0就是 UP
- 如果是1就是 Down 不能将 `Level-1` 计算出来的路由放入到 `Level-2` 的 LSP 中进行携带
<font color="#ff0000">收到泄露路由的设备就不会再把这个路由发给其他设备</font>
#### 1. `Level-1` > `Level-2` ：在 AR1和 AR4的环回口配置同一个 IP 地址
思路：让 AR2/3判断走 `Level-1` 还是 `Level-2` 学到的路由

配置命令：
```

 ip address 1.1.1.1 255.255.255.255 
 isis enable 1
```

<font color="#ff0000">接口级认证配置的优先级高于区域级认证配置</font>，所以启用的是 `G0/0/1` 下配置的认证方式和密钥 `Huawei@1`
### 3. 密文认证（auth-type = 2）
#### 优点
1. 加密
2. 防重放攻击-sequence 递增来防止重放

#### 配置方法
##### 第一种：接口直配式 OSPF 加密认证
接口下配置：`[Huawei-GigabitEthernet0/0/1]ospf authentication-mode md5 88 plain Huawei@123`
- `md5`：认证算法为 MD5；
- `88`：密钥 ID（仅用于标识该密钥，两端设备接口的密钥 ID 必须一致）；
- `plain Huawei@123`：明文密钥串（设备会自动加密存储），两端密钥串必须完全一致。
	![](DICM/Pasted%20image%2020251212165919.png)

##### 第二种：基于 Keychain 的 OSPF 加密认证
1. 绝对时间模式
```
[Huawei]keychain abc mode absolute  // 全局定义密钥链abc，采用绝对时间模式
[Huawei-keychain-abc]key-id 1       // 密钥ID 1
[Huawei-keychain-abc-keyid-1]algorithm md5  // 算法MD5
[Huawei-keychain-abc-keyid-1]key-string plain Huawei@123  // 密钥串
[Huawei-keychain-abc-keyid-1]send-time utc 00:00 2025-12-12 to 00:00 2026-01-12  // 发送密钥的有效期
[Huawei-keychain-abc-keyid-1]receive-time utc 00:00 2025-12-12 to 00:00 2026-01-12  // 接收密钥的有效期
[Huawei-keychain-abc-keyid-2]algorithm md5  // 密钥ID 2
[Huawei-keychain-abc-keyid-2]key-string plain k3s  // 另一组密钥串
// （可省略time，默认永久有效；也可配置与key-id 1不同的有效期）

// 关键步骤：将密钥链绑定到OSPF接口
[Huawei-GigabitEthernet0/0/1]ospf authentication-mode keychain abc
```

2. 周期模式
```
[Huawei]keychain abc mode periodic // 定义密钥链abc为周期模式 
[Huawei-keychain-abc]key-id 1 // 密钥ID 1（与对端一致） 
[Huawei-keychain-abc-keyid-1]algorithm md5 // 认证算法 
[Huawei-keychain-abc-keyid-1]key-string plain Huawei@123 // 密钥串 
// 核心：配置周期生效时间（三选一：daily/weekly/monthly） 

[Huawei-keychain-abc-keyid-1]send-time weekly 0 00:00 to 6 23:59 // 发送密钥的周期 
[Huawei-keychain-abc-keyid-1]receive-time weekly 0 00:00 to 6 23:59 // 接收密钥的周期 
[Huawei-keychain-abc]key-id 2 // 密钥ID 2（备用/轮换） 
[Huawei-keychain-abc-keyid-2]algorithm md5 
[Huawei-keychain-abc-keyid-2]key-string plain k3s 
[Huawei-keychain-abc-keyid-2]send-time weekly 0 00:00 to 6 23:59 // 可配置不同周期 
// 绑定到OSPF接口 
[Huawei-GigabitEthernet0/0/1]ospf authentication-mode keychain abc
```

## 2、路由的汇总-ABR  ASBR
### 1. <font color="#ff0000">区域内无法汇总</font>
### 2. 区域间路由的汇总
#### 1. 目的
1. 减少其他区域的 LSDB 的规模
2. 减少其他区域的路由表规模
3. 节约其他区域的路由器的性能开销
	- 当明细路由震荡的时候，不会影响其他区域路由器的 PRC 计算
	- 只有开销值最大的那条明细震荡或者所有的明细消失，才会影响其他区域的 PRC 计算
#### 2. 配置
![](DICM/Pasted%20image%2020251216153420.png)

在 <font color="#ff0000">ABR</font> 设备上配置：
```
ospf 1
 area 0.0.0.1 
 abr-summary 192.168.0.0 255.255.252.0
 
用于防环
[Huawei]ip route-static 192.168.0.0 255.255.0.0 NULL 0
```
在 AR5上：
```
 将接口加入 OSPF 域
interface LoopBack10
 ip address 192.168.0.1 255.255.255.0 
 ospf cost 5
 ospf network-type broadcast
interface LoopBack11
 ip address 192.168.1.1 255.255.255.0 
 ospf cost 10
 ospf network-type broadcast
interface LoopBack12
 ip address 192.168.2.1 255.255.255.0 
 ospf cost 15
 ospf network-type broadcast
interface LoopBack13
 ip address 192.168.3.1 255.255.255.0 
 ospf cost 20
 ospf network-type broadcast
 
 宣告网络
ospf 1 router-id 5.5.5.5 
import-route direct
network 192.168.0.0 0.0.0.255 
network 192.168.1.0 0.0.0.255 
network 192.168.2.0 0.0.0.255 
network 192.168.3.0 0.0.0.255 
```
#### 3. 通过路由汇总来进行过滤
在汇总的后面加上 `not-advertise` 就可以<font color="#ff0000">屏蔽这条路由</font>
```
ospf 1 
area 0.0.0.1 
abr-summary 192.168.3.0 255.255.255.0 not-advertise
```

#### 4. 注意事项
1. `abr-summary` 是在 ABR 设备上使用的，在其他设备上配置不起效果
2. `asbr-summary` 同理是在 ASBR 设备上使用的，只在引入外部路由后，用来汇总外部路由
3. 通过路由汇总来进行过滤也是同理，有 `abr-summary x.x.x.x mask not-advertise` 和 `asbr-summary x.x.x.x mask not-advertise`

如果 AR3和 AR4都配置 `abr-summary 192.168.0.0 255.255.252.0`
此时：

## 3、路由过滤--过滤路由、过滤 LSA
1. 汇总方式+ `not-advertise`（很少用）
2. 过滤路由
3. 过滤 LSA3/5/7
4. 接口下过滤 LSA（可以过滤所有）


### 1. 过滤路由
如果在 IR 上仅让 IR 自己的路由表过滤，但区域内内的 LSA 正常泛洪 ABR 传过来的路由，<font color="#ff0000">同区域的其他设备路由表不受影响</font>
如果在 ABR 上过滤区域内或者区域间的路由，那么根据矢量特性，就不会往其它区域产生 LSA3
配置命令：
```
ospf 1
filter-policy ip-prefix xxx import 
```
注意：ACL 和前缀列表的对比
1. ACL 和前缀列表都可以用于匹配路由，但是 ACL 不能匹配掩码，而前缀列表可以匹配掩码，但是 ACL 可以匹配一些特殊路由，而前缀列表不行
	例如，匹配奇数路由，偶数路由
2. ACL 可以用于包过滤，NAT，QOS 等等，但是前缀列表只能用于匹配路由
	华为 ACL 只能用基本 ACL 匹配路由，不能使用高级 ACL。

### 2. 过滤 LSA3/5/7

对于 LSA5和 LSA7：<font color="#ff0000">NSSA 区域的 ABR（ASBR）但是不能过滤自己 LSA7转 LSA5的路由</font>

#### 配置

#### LSA3：ABR 上区域下配置
```
area 0.0.0.0/1
filter ip-prefix xxxx export/import
```
#### LSA5和 LSA7：<font color="#7030a0">ASBR</font> 的<font color="#ff0000">进程下的出方向配置</font>
	进程的入方向过滤是过滤路由，出方向过滤是过滤LSA

```
ip ip-prefix 12 index 10 deny 12.1.1.0 24
ip ip-prefix 12 index 20 permit 0.0.0.0 0 less-equal 32

ospf 1 router-id 3.3.3.3 
 filter-policy ip-prefix 12 export
```
#### 注意 import 和 export
- **`import`**：过滤 **“从其他区域 / 外部传入本设备”的路由（比如 ABR 接收其他区域的 LSA3，或设备接收 OSPF 路由）。
    简单记：**“进设备的路由，用 import”**。
- **`export`**：过滤 **“从本设备发布到其他区域 / 外部”的路由（比如 ABR 向其他区域发布 LSA3，或设备向 OSPF 域发布路由）。**
	**简单记：**“出设备的路由，用 export”**。

### 3. 接口下过滤 LSA，可以过滤所有 LSA 类型

- 产生的原因：设备性能不够

使用场景：
1. 末节点设备性能不行，配置接口过滤 LSA，然后自己配置一条缺省的静态，访问其它网络
2. 过滤高速链路的 LSA
	![](DICM/Pasted%20image%2020251217151133.png)
	在 AR2与 AR5直连的 `GE4/0/0` 端口配置过滤
	- 让这条 100G 高速链路的 LSA 信息，不扩散到其他设备（AR1、AR3、AR4），只在 AR2 和 AR5 之间生效。
	- AR2 和 AR5能直接通过这条 100G 链路通信
	- 对其他设备（AR1、AR3、AR4）的影响：
		1. 这些设备的 OSPF LSDB 中，不会收到 “AR2-AR5 100G 链路” 的 LSA 细节（比如这条链路的速率、开销、接口信息）。
		2. 但它们依然能通过 “AR2-AR3-AR5”“AR2-AR4-AR5” 这些 1G 链路的 LSA，感知到 AR5 的可达性 
		3. 只是在选路时，会优先选 “AR2-AR5 100G 链路”（因为高速链路本身 Metric 小，即使 LSA 没扩散，设备通过直连 / 其他路径的 LSA 能间接感知到这条最优路径）
	 实际作用：
	- 减轻 AR1、AR3、AR4 的负担：这些设备的 LSDB 不会被 “100G 链路的 LSA” 占用内存，SPF 计算也不用处理这条链路的细节；
	- 不影响主路径：AR2 和 AR5 之间的 100G 链路依然是主用（最优）路径，通信效率不受影响；
	- 备用路径仍可用：如果 100G 链路故障，其他设备的 LSDB 里有 1G 链路的 LSA，能自动切换到备用路径。



#### 实验1：过滤全部 LSA，配置静态路由
在 AR3的 `GE0/0/0` 端口配置 `ospf filter-lsa-out all`
在 AR4配置静态路由指向外侧

1. 在配置 `ospf filter-lsa-out all` 后，所有的 LSA 都会被过滤，但是在配置<font color="#ff0000">之前的不会消失，要等待老化</font>
2. 如图所示：

## 4. 默认路由
### 1.强制下发缺省路由
计算节点<font color="#ff0000">没有</font>其它 OSPF 进程或者其它路由协议计算出默认，然后自己强制产生 LSA5的缺省。<font color="#d83931">用于连接总部，总部可能存在成百上千条路由</font>。
#### 配置
```
ospf 1

default-route-advertise always 
```

### 2.非强制下发缺省路由
计算节点<font color="#ff0000">必须要有</font>其它 OSPF 进程或者其它路由协议学习到的默认并放入到路由表，然后自己才会产生 LSA5的缺省。
#### 配置
```
ospf 1

default-route-advertise
```

##### 原因
- **AR1（Area 0）的 1.1.1.1**：是 Area 0 的内部路由，通过 Type 3 LSA 传递到其他区域。
- **AR4（Area 1，Stub 区域）**：Area 1 是 Stub 区域，所以 AR4 收到的 “1.1.1.1” 会被标记为 **Stub 类型**（符合 Stub 区域的路由表示规则）。
- **AR5（Area 2，普通区域）**：Area 2 是普通区域，所以 AR5 收到的 “1.1.1.1” 会被标记为 **Inter-area（区域间路由）**（这是普通区域对 Type 3 LSA 的默认标识）

##### 原因
如果在 AR2设置过滤，那么也就是说当 AR4想访问 AR1的 `1.1.1.1`，那么需要实际上 AR3上有对应的路由，不然即使要访问，那么也会因为发给 AR3的时候，因为 AR3没有 `1.1.1.1` 的路由而被丢弃


#### 4. 新加入一个设备，并配置区域，再把 AR3与 AR4的开销改为1000
![](DICM/Pasted%20image%2020251217165218.png)
此时可以看到，AR4访问 AR1的 `1.1.1.1` 的下一跳变为 AR6所在端口
![](DICM/Pasted%20image%2020251217165305.png)

##### 原因
1. 虚链路（VLink）的作用是**让不连续的 Area 0 通过非骨干区域（这里是 Area 1）“逻辑连通”**，但虚链路本身是一条 **“逻辑链路”**，它的 Cost 是**实际物理路径的 Cost 总和 （不是单独的链路 Cost）**
2. 把 AR4 的 GE0/0/0 端口 Cost 改成 1000 后，**虚链路的实际路径 Cost 被大幅拉高**，而另一条路径的 Cost 更低，因此 OSPF 会选 Cost 小的路径



### 3. VLINK 的特性
1. 特殊区域不能做 VLINK
2. 骨干区域不能做 VLINK
3. VLINK 不传递 LSA5

### 4. VLINK 的防环
1. 区域0的路由不会汇总进做虚链路的区域
2.   从 VLINK 收到的 LSA1/2不会生成发往做 VLINK 区域的 LSA3
	 从 VLINK 收到的 LSA3也不会生成发往做 VLINK 区域的 LSA3
	- 这里 VLINK 指的是做 `vlink-peer` 的路由器，VLINK 区域是整个 `Area` 区域

### 5. VLINK 配置不当导致的环路
在 AR3和 AR4之间配置虚链路 `VLINK`
![](DICM/Pasted%20image%2020251218121109.png)

#### 配置要点
在 AR3一开始的配置是：
```
ospf 1 router-id 3.3.3.3 
 area 0.0.0.0 
  network 13.1.1.3 0.0.0.0 
  network 23.1.1.3 0.0.0.0 
```
配置 VLINK 后要把配置改成：（把 `network 23.1.1.3 0.0.0.0` 划分到 Area1，<font color="#ff0000">让它从 IR 设备变为 ABR 设备</font>）
```
ospf 1 router-id 3.3.3.3 
 area 0.0.0.0 
  network 13.1.1.3 0.0.0.0 
 area 0.0.0.1 
  network 23.1.1.3 0.0.0.0 
  vlink-peer 4.4.4.4
```

只有把 AR3 的`23.1.1.3`转移到 Area1，AR3 才会成为同时归属 Area0 和 Area1 的 ABR，且和 AR4 共享 Area1 这个传输区域，虚链路才能正常建立。

AR2：也要把 `network 23.1.1.2 0.0.0.0 ` 放到 `Area1`
```
ospf 1 router-id 2.2.2.2 
 area 0.0.0.0 
  network 12.1.1.2 0.0.0.0 
 area 0.0.0.1 
  network 23.1.1.2 0.0.0.0 
  network 24.1.1.2 0.0.0.0 
```

AR2 把`23.1.1.2`宣告进 Area 1，是为了**在 Area 1 内建立完整的中转拓扑**，让 AR3 和 AR4 的虚链路报文能通过 AR2 转发



在 AR5引入路由
```
ip route-static 5.5.5.5 255.255.255.255 NULL0
ospf 1 router-id 5.5.5.5 
 import-route static
```

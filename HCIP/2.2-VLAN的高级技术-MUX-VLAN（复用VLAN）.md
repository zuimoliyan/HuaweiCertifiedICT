## 一、产生背景和配置
![](DICM/Pasted%20image%2020251129154949.png)
<font color="#6425d0">PC5、PC6</font>主机代表公共资源（服务器、打印机），需要其它的 VLAN 能够互访。
<font color="#ff0000">注意：接口可以划分到主 VLAN，但是不能配置 VLANIF</font>


## 二、配置命令

![](DICM/Pasted%20image%2020251129161050.png)
### 1. 配置主 VLAN 和从 VLAN
```
vlan 100
 mux-vlan
 subordinate separate 10  //隔离型从VLAN
 subordinate group 20  //互通型从VLAN
```
### 2. 将接口划入相应的 VLAN
```
interface GigabitEthernet0/0/1
 port link-type access
 port default vlan 10
 
 interface GigabitEthernet0/0/3
 port link-type access
 port default vlan 20
 
 interface GigabitEthernet0/0/6
 port link-type access
 port default vlan 100

不全写了
```
### 3. 接口下配置启用 MUX-VLAN
```
interface GigabitEthernet0/0/1
 port mux-vlan enable
interface GigabitEthernet0/0/2
 port mux-vlan enable
interface GigabitEthernet0/0/3
 port mux-vlan enable
interface GigabitEthernet0/0/4
 port mux-vlan enable
interface GigabitEthernet0/0/5
 port mux-vlan enable
interface GigabitEthernet0/0/6
 port mux-vlan enable
```
## 三、注意点补充
### 1. 主 VLAN 的 MAC 地址会复制到从 VLAN，从 VLAN 也会复制到主 VALN
如[配置命令](#二、配置命令)下展示的，此时 VLAN10、20下的 MAC 地址都会复制一份到 VLAN100，VLAN100下的也会复制一份到 VLAN10、20

## 四、实验案例
### 案例1：[简单的MUX-VLAN配置](#二、配置命令)
### 案例2：在案例1的基础上配置 MUX-VLAN 访问外网
1. `聚合VLAN` 可以配置 `VLANIF`，但是不能划分物理接口进入该 `VLAN`
2. `MUX-VLAN` 不能配置 `VLANIF`，但是可以划分物理接口进入该`VLAN`
	补充：<font color="#ff0000">高版本的 MUX-VLAN 可以配置 VLANIF</font>，如果是低版本的<font color="#7030a0">可以将网关接入到主 VLAN 的对应接口</font>

解决办法：引入一个路由器来充当网关，并在交换机与路由器直连的端口配置<font color="#ff0000">将这个端口加入主 VLAN</font>
![](DICM/Pasted%20image%2020251129163018.png)
```
interface GigabitEthernet0/0/10
 port link-type access
 port default vlan 100
 port mux-vlan enable
```

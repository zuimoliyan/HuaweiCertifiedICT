## 整体思路
**双向引入**：R2、R3 都要做两件事 ——① 把 OSPF 路由引入 ISIS；② 把 ISIS 路由引入 OSPF
**双点冗余**：两台边界设备互为备份，提升网络可靠性，但也带来了路由循环的风险
### 防环
防环的核心手段是 `Tag标记 + 路由策略过滤`
### 次优
优化次优路由的核心思路是 `差异化开销 + 路由优先级`，让设备 “聪明地” 选择更近的路径

## 3、解决 AR3访问 `5.5.5.5` 的次优路径问题
在 AR2上 ISIS 下配置：` import-route ospf 1 inherit-cost`
在 AR3上配置：
```
route-policy isis permit node 10 
 if-match ip-prefix 6 
 apply preference 200 
isis
  preference route-policy isis
```
OSPF 的次优同理：
```
default cost inherit-metric
```

## 4、解决环路问题
在 AR2、AR3的 OSPF 场景：
```
route-policy abc deny node 10  # 第一步：阻断已标记的路由
 if-match tag 10               # 匹配“跨域引入过的路由”（带Tag 10）
#
route-policy abc permit node 20  # 第二步：放行原生路由并标记
 apply tag 10                    # 给“首次跨域引入的路由”打Tag，做身份标识
#
ospf
 import-route isis 1 type 1 route-policy abc  # 引入时执行上述策略
```
在 AR2、AR3的 ISIS 场景：
```
route-policy abc deny node 10  # 第一步：阻断已标记的路由
 if-match tag 10               # 匹配“跨域引入过的路由”（带Tag 10）
#
route-policy abc permit node 20  # 第二步：放行原生路由并标记
 apply tag 10                    # 给“首次跨域引入的路由”打Tag，做身份标识
#
isis
 import-route ospf 1 inherit-cost route-policy abc 
```
**Tag 标记跨域路由 + 拒绝已标记的路由再次引入**，从源头阻断环路

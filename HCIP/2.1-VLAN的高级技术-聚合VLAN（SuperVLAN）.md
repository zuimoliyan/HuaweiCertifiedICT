## 一、产生背景
1. IP 地址的浪费（划分多个子网，每个子网少两个地址（子网广播地址和网络地址））
2. VLANIF 接口的浪费
3. 配置繁琐-配置相应的网关，发布到路由协议中
4. 配置多个 VLAN

## 二、VLAN 聚合的原理
管理员根据业务需求（楼层或者部门），去划分相应的部门用来隔离广播域，为了节约 IP 地址，或者简化配置，可以启用一个超级 VLAN，超级 VLAN 可以将所有的 `Access VLAN` 聚合在一起，然后在 superVLAN 配置 VLANIF 和 IP 地址
1. <font color="#ff0000">物理接口不能属于超级（聚合）VLAN</font>
2. <font color="#ff0000">超级 VLAN 下的accessVLAN不能配置 VLANIF</font>

## 三、相关配置
### 案例1：链路聚合的基础配置实验
![](DICM/Pasted%20image%2020251129143115.png)
1. 先正常配置划分好 VLAN
2. <font color="#ff0000">然后再在交换机下配置 superVLAN</font>
	```
	vlan 100
	 aggregate-vlan  //创建superVLAN
	 access-vlan 10 20 30  //将VLAN10、20、30划入超级VLAN
	 #
	interface Vlanif100   //用于与外界通信，当做网关
	 ip address 192.168.1.254 255.255.255.0
	```
3. 创建一个与外界直连的 VLAN，配置上地址，用于与外界通信
	```
	interface GigabitEthernet0/0/9  //这个接口与外网直连，所以配置将接口划入vlan1000
	 port link-type access
	 port default vlan 1000
	#
	interface Vlanif1000  //配置好vlanif与外接通信
	 ip address 100.1.1.1 255.255.255.0
	```
4. 再在交换机和外界路由器分别配置默认路由
```
交换机
ip route-static 0.0.0.0 0.0.0.0 100.1.1.2

路由器
ip route-static 192.168.1.0 24 100.1.1.1
```
#### 补：如果想实现 vlan 间通信呢？----VLAN 间的 ARP 代理
与 [1-以太交换安全](1-以太交换安全.md)中 `开启 ARP 的 VLAN 内代理` 可以对比配置
还是以上图为例：在 superVLAN 下配置 `arp-proxy inter-sub-vlan-proxy enable` 即可实现同网段下不同 VLAN 通信
```
interface Vlanif100
 ip address 192.168.1.254 255.255.255.0
 arp-proxy inter-sub-vlan-proxy enable
 ```
 
## 四、相关命令
1. VLAN 接口下配置： `aggregate-vlan`：将已创建的 VLAN 指定为 `SuperVLAN`
2. VLAN 接口下配置： `access-vlan 10 20 30`：将`VLAN10、20、30`划入超级VLAN 作为子 VLAN

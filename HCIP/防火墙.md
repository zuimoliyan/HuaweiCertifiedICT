# 一、什么是防火墙（FW/FireWall）
## 1、定义
组网设备，可以支持二层和三层转发。防火墙具有更加丰富的控制能力。
	防火墙可以实现：<font color="#ff0000">安全策略</font>（升级版的包过滤）、带宽策略、黑名单、内容的安全检测模块
	防火墙对 VPN 的支持更加丰富（GRE、L2TP、IPSEC、SSL、MPLS、VXLAN），基于应用和用户来实现精细控制。

# 二、硬件防火墙按照控制来分类

## 1、包过滤防火墙
1. 定义：通过 ACL 来控制流量，先后（同一条流量或者镜像流量）的报文没有关联，每个包到达包过滤防火墙的时候都要逐一检查，消耗性能。配置量非常大，状态检测提供<font color="#ff0000">会话表</font>。

2. 包过滤的缺点
	- 配置非常繁琐，包过滤 FW 无法关联后续链路
	- 每个报文都需要检查，效率低，消耗 CPU
	- 不能识别应用
	- 无法适应多信道（看 FTP）
	回顾 ACL
	- 基本 ACL：只能控制源 IP 网段
	- 高级 ACL：可以控制源目 IP 网段、以及传输层信息。无法识别/控制 APP 信息
## 2、代理防火墙
工作在 **OSI 七层模型的应用层**的网络安全设备，它的核心作用是**作为内部网络和外部网络之间的 “中间人”**，代替内部主机与外部服务器建立连接，以此隔离并保护内部网络
- 外部服务器无法直接识别内部主机的 IP 地址
- 代理防火墙会对**应用层的具体内容**进行深度检测，比如 HTTP 协议的请求头、URL、报文内容，FTP 的传输文件类型等
![[DICM/Pasted image 20260118170912.png]]

1. 优点： 检测效果最好的，但是防火墙需要维护两个对话（客户端和防火墙、防火墙和服务器），需要深包分析（读到应用层）
2. 缺点：防火墙会成为转发的瓶颈
## 3、状态检测防火墙
1. 定义：通过防火墙的流量会记录会话信息（五元组），后续的流量（同一条流或者镜像流量）只要匹配会话，就可以转发，工作效率高

2. 首包创建会话
	1. 建立TCP 握手的第一个报文（SYN 置位的报文）
	2. ICMP 的 request 报文
	3. 除了以上两种之外的所有单播报文（例如 UDP 报文和 OSPF 报文）

# 三、安全区域

### 1.华为防火墙预定义的安全区域
1. local：防火墙自身的接口都属于 local 区域，不需要添加接口到 local。
2. Trust 区域（`priority = 85`）：一般用于连接内网
3. untrust 区域（`priority = 5`）：一般用于连接外网，Internet 接口，或者 ISP 接口
4. DMZ 区域（`priority = 50`）：一般用于连接服务器
### 2. 自定义安全区域
可以配置96个

### 安全策略--不仅仅是五元组，可以匹配应用、用户和服务
1. 条件--如果是不同条件字段（如下面的九项），那么是<font color="#ff0000">与</font>的关系；如果是相同条件字段，那么是<font color="#ff0000">或</font>的关系
	1. 源安全区域
	2. 目的安全区域
	3. 源 IP 网段
	4. 目的 IP 网段
	5. DSCP
	6. 时间段
	7. 应用
	8. 用户
	9. 服务
2. 动作
	1. 拒绝 deny
	2. 允许 permit
3. 安全规则
	1. 匹配的时候从上往下匹配，配置的时候明细的策略写在前面，粗的策略写在后面



### 不受安全策略控制的报文
1. 默认属于同一个安全区域的网络互访不受控制---可以形成会话表
2. 组播和广播不受安全策略控制---不会形成会话表
3. 开起来网管功能的接口不受安全策略控制---可以形成会话表
4. 命中了会话表的镜像流量
5. 开启了 ASPF（指定应用包过滤）功能，匹配了 `server-map` 的流量---可以创建会话
	- ASPF 会维护一个会话状态表，记录每个连接的源地址、目的地址、端口号、协议类型、连接状态等信息
	- 当数据包通过防火墙时，先匹配会话表，只有匹配已建立的合法会话或符合协议状态的数据包才会被放行，反向回应报文也能基于会话表动态允许通过，无需手动配置大量静态规则
6. 配置了 `undo firewall packet-filter basic-protocol enable`，那么对协议报文不做安全策略检查



# 四、NAT

1. no-pat：只对 IP 地址做转换，不对端口号做转换
	- 缺点：公网 IP 和私网 IP 是一对一映射，不能节省 IP 地址
	- 优点：产生 server-map，正向的 server-map 能使<font color="#ff0000">后续的报文直接命中而不需要进行 NAT 策略匹配</font>，节约了性能；反向的 server-map，只要配置了安全策略放行了，那么可以让外网访问内网
2. NAPT：只对 IP 地址和端口号一起转换，公网 IP 和私网 IP 可以实现1：N。可以节约 IP 地址，不会产生 server-map 表
3. full-cone NAT：完全圆锥型 NAT（一对一 NAT、静态 NAT），它将同一内网 IP + 端口的所有请求映射到固定的公网 IP + 端口，且任意外部主机可通过该公网 IP + 端口主动发起连接到内网主机，是 NAT 中最开放的类型
4. smart-nat
5. Easy-IP
6. nat-server



# 五、NAT 的种类

![[DICM/Pasted image 20260201175244.png]]
以 `192.168.1.1:1027` (私网地址和端口号)---> `100.1.12.1:1028` (NAT 转换对接的公网地址和端口号)--> `8.8.8.8` (私网要访问的地址和端口号)

- 1（私网）：1（公网）NAT：no-pat 模式
- N（私网）：1（公网）NAT
	1. 对称型 NAT（默认）：NAPT 模式
	2. 圆锥形 NAT
		1. `Endpoint-and-port-dependent`：IP和端口受限的NAT
			- 理解：该NAT要求回包的 IP 和端口号都必须一样才可以进行NAT转换
			- 场景：私网192访问公网8.8.8.8设备，当8.8.8.8要回包给192时，8.8.8.8必须要用自己8.8.8.8:21的这个IP这个端口回包给100.1.12.1才能映射出192这个地址，包才能发回给192
		2. `Endpoint-dependent`：IP受限的NAT
			- 理解：该NAT要求回包的IP地址相同，端口号多少都可以
			- 场景：192访问公网`8.8.8.8`设备，当`8.8.8.8`要回包给192时，只要回包给 `100.1.12.1` 的源 IP 地址是 `8.8.8.8` 即可，端口号可以是随机的
		3. `Endpoint-independent`：完全圆锥型NAT
			- 理解：该NAT不要求回包的IP和端口号一致
			- 情景：192.168.1.1访问公网 `8.8.8.8` 设备经过 `100.1.12.1:1028` 转到后，**无论公网哪个设备、哪个端口**，只要向 NAT 的公网地址 + 端口（ `100.1.12.1:1028` ）发送数据包，NAT 设备都会直接把数据包转发到内网的 `192.168.1.1:1027`，**不会检查发数据包的公网设备是不是内网之前访问过的那个**。


# 六、策略路由（Policy-Based Routing）

使得网络设备不仅能够基于报文的目的IP地址进行数据转发，更能基于其他元素进行数据转发，例如源IP地址、源MAc地址、目的MAc地址、源端口号、目的端口号、VLAN-ID等等

## 案例1：简单的策略路由实验

描述：通过 AR1 `telnet` 来访问 AR2和 AR3的环回口 `8.8.8.8` 模拟远程登录
![[DICM/Pasted image 20260201192905.png|750]]

默认配置完毕后，当在 AR1上使用 `telnet 8.8.8.8`，会默认连接到 AR3，<font color="#ff0000">此时我们希望通过策略路由让他连接到 AR2</font>
	![[DICM/Pasted image 20260201193147.png]]

AR2和 AR3：（这个是可以 telnet 的前提）
```
user-interface vty 0 4
 authentication-mode password 123
 user privilege level 15
```

FW1：配置接口地址、加入 zone、配置 easy-IP 的 NAT、配置策略路由
```
firewall zone trust
 add interface GigabitEthernet1/0/1
#
firewall zone name isp1
 set priority 10
 add interface GigabitEthernet1/0/2
 
firewall zone name isp2
 set priority 11
 add interface GigabitEthernet1/0/3
#
security-policy
 default action permit
#
nat-policy
 rule name nat
  source-zone trust
  destination-zone isp1
  destination-zone isp2
  source-address 192.168.1.0 mask 255.255.255.0
  action source-nat easy-ip

# 策略路由
policy-based-route
 rule name tel
  source-zone trust
  source-address 192.168.1.0 mask 255.255.255.0
  destination-address 8.8.8.8 mask 255.255.255.255
  service telnet
  action pbr egress-interface GigabitEthernet1/0/2 next-hop 100.1.12.2
```

# 七、防火墙配置实验

## 基本实验

理解**基于区域**和**基于具体 IP 地址**两种匹配方式的不同
1. 基于区域
	- 策略只关注 “数据从哪个区域来、到哪个区域去”，不限制具体的 IP 地址
	- 覆盖 `trust` 区域内**所有 IP**访问 `untrust` 区域**所有 IP**的流量，范围非常宽泛
2. 基于具体 IP 地址
	- 策略只关注 “数据的源 IP 和目的 IP”，不限制所属区域
	- 仅覆盖指定 IP 段之间的流量，范围精准
	- 适合 “精准放行” 的需求，比如允许内网某网段访问特定服务器网段

### 案例1：简单的 FTP 防火墙配置实验（source-zone 模式）

如图所示，我们希望无论是 FTP 的<font color="#ff0000">主动连接还是被动连接</font>都可以进行通信
	![[DICM/Pasted image 20260120174827.png]]
1. 将防火墙与 `Client` 和 `server` 的接口配置对应的 IP 地址
2. 将与 `Client` 相连的接口区域划入 `trust` 区域，将与 `server` 相连的接口区域划入 `untrust` 区域
```
firewall zone trust
 add interface GigabitEthernet1/0/1
#
firewall zone untrust
 add interface GigabitEthernet1/0/2
```
3. 配置安全策略
```
security-policy 
rule name FTP_Trust_to_Untrust					#安全策略的名字是“FTP_Trust_to_Untrust” 
source-zone trust 								# 源区域 
destination-zone untrust 						# 目标区域 
source-address 192.168.1.0 mask 255.255.255.0   # 源IP段 
service ftp 									# 限定FTP协议（21端口） 
action permit 									# 允许
```
#### 注意
上面的安全策略是：只有满足<font color="#ff0000">「从 trust 区域的接口进来」</font>并且<font color="#ff0000">「源 IP 是 192.168.1.0/24」</font>并且<font color="#ff0000">「是 FTP 流量」</font>的数据包，<font color="#6425d0">才能被允许通行</font>

## NAT 实验
### 案例1：NAPT 实验

如图，我们希望通过防火墙让 `Client1` 以 NAPT 方式访问 AR1
	![[DICM/Pasted image 20260121182345.png|750]]

FW1的配置
```
# 这里是正常配置接口的IP地址
interface GigabitEthernet1/0/1            
 undo shutdown                            
 ip address 192.168.1.254 255.255.255.0                                         
interface GigabitEthernet1/0/3            
 undo shutdown                            
 ip address 200.1.1.1 255.255.255.0  
     
# 将指定接口加入trust区域
firewall zone trust                                                
 add interface GigabitEthernet1/0/1       
 
# 将指定接口加入untrust区域
firewall zone untrust                                              
 add interface GigabitEthernet1/0/3
 
#配置nat的地址池和模式
nat address-group 1 
 mode pat                                 
 section 0 200.1.1.10 200.1.1.20
 
# 安全策略：默认允许所有流量
security-policy                           
 default action permit         # 默认是拒绝，不配置则内网无法访问外网

# 配置NAPT策略
nat-policy                                
 rule name NAPT                           
  source-zone trust                       
  destination-zone untrust                
  source-address 192.168.1.0 mask 255.255.255.0
  action source-nat address-group 1         # 配置规则动作为源NAT，使用编号1的地址池进行转换
```

<font color="#ff0000">LSA4告诉路由器"ASBR 在哪里"，LSA5告诉路由器"外部网络怎么走"，两者配合完成外部路由的完整通告过程</font>

# 一、LSA5--自治系统外部 LSA
## 1、相关摘要和注意事项

1. 内容：当引入了<font color="#ff0000">其他路由协议生成</font>的路由或者<font color="#ff0000">其他 OSPF 进程</font>的路由（路由表要有）
2. Link-State ID：就是外部路由的网络信息
3. 泛洪范围：再整个 OSPF domain 中泛洪的（除了特殊区域），<font color="#ff0000">五类泛洪并不需要有区域0</font>
4. 通告者：ASBR（有能力产生 LSA5或者 LSA7的路由器，但有可能并没有产生），如果该设备是 ASBR 的话，那么在自己的 LSA1的 Flags 中的 `E=1`
	![](DICM/Pasted%20image%2020251210091519.png)

## 2、计算成路由
### 1. 计算节点和 ASBR 在同一个区域：LSA1+LSA2（可选）+LSA5
1. 通过 LSA5获取网络号和掩码
2. 通过 LSA5获取通告路由器的 RID，然后将去往 ASBR 的下一跳作为外部路由器的下一跳
3.  `import-route static type 1`，默认情况下，OSPF 使用 Type 2 作为外部路由的度量值类型
	- 如果 metric-type = 2（默认就等于2），计算开销的时候就仅计算外部开销，不计算内部开销
	- 如果 metric-type = 1，计算开销的时候内部和外部开销都要计算

### 2.计算机点和 ASBR 在不同区域：LSA1+LSA2（如果点到点就没有）+LSA4+LSA5
1. 通过 LSA5获取网络号和掩码
2. 通过 LSA5获取通告路由器的 RID，然后将去往 ASBR 的下一跳作为外部路由器的下一跳（<font color="#ff0000">但是没办法直接获取 ASBR 的下一跳</font>）

	如果metric-type=1，计算开销的开销的时候计算外部开销和内部开销（到ASBR的距离）之和

## 3、FA--转发地址
### FA 产生的背景：为了解决 MA 网路的次优问题
	路由器没有选择“最短”或“最快”的路径到达目的（网段/路由）

### 1.FA = 0, metric-type = 1
外部路由的下一跳指向 ASBR---->>下一跳指向 ABR
开销 = 外部开销+去往 ASBR 的内部开销

### 2.FA != 0
外部路由的下一跳指向 FA（FA 对应的 OSPF 的内部路由的下一跳）
开销 = 外部开销+去往 FA 对应的 OSPF 路由的开销

FA 不等于0的条件：
1. 指向外部路由的接口宣告进了OSPF
2. 该接口不是P2P和P2MP的网络类型（broadcast）
3. 该接口没有被静默
4. FA 对应的路由必须是 OSPF 的内部路由（区域内和区域间）
### 3. FA!=0，通过 LSA5计算外部路由
1. 计算节点和 ASBR 在同一个区域：LSA1+LSA2（可选）+LSA5
	1. 通过 LSA5获取网络号和掩码
	2. 通过 LSA5获取通告路由器的 RID，然后将去往 FA 的下一跳作为外部路由器的下一跳
	3.  `import-route static type 1`，默认情况下，OSPF 使用 Type 2 作为外部路由的度量值类型
		- 如果 metric-type = 2（默认就等于2），计算开销的时候就仅计算外部开销，不计算内部开销
		- 如果 metric-type = 1，计算开销的时候内部（计算节点到 FA 对应OSPF 内部路由的距离）和外部开销都要计算
2. 计算机点和 ASBR 在不同区域：LSA1+LSA2（如果点到点就没有）+LSA4+LSA5
	1. 通过 LSA5获取网络号和掩码
	2. 通过 LSA5获取通告路由器的 RID，然后将去往 FA 的下一跳作为外部路由器的下一跳（<font color="#ff0000">但是没办法直接获取 ASBR 的下一跳</font>）
	3. 通过 LSA3获取去往 FA的下一跳
	4. 如果metric-type=2（默认），计算开销的开销的时候仅计算外部开销，不计算内部开销
	   如果metric-type=1，计算开销的开销的时候计算外部开销和内部开销（到ASBR的距离）之和

# 二、LSA4--ASBR summary
## 1、相关内容
1. 内容：ASBR 的 RID 和 ABR 到 ASBR 的距离
2. Link-State： 是ASBR ID
3. 泛洪范围：一个区域内泛洪
4. 通告者：ABR

# 三、注意点
## 1. 有 LSA4不一定有 LSA5
LSA4 和 LSA5 的产生条件不同，它们之间没有必然的依赖关系：
- **LSA4 的产生条件**：只要存在 ASBR（自治系统边界路由器），且该 ASBR 与接收 LSA5 的路由器不在同一个区域，ABR 就会产生 LSA4 来通告 ASBR 的位置信息。
- **LSA5 的产生条件**：ASBR 通过重分发引入了外部路由（如静态路由、RIP、BGP 等），才会产生 LSA5 来通告这些外部路由。

**常见场景**：
1. **有 LSA4 无 LSA5**：ASBR 存在，但尚未引入任何外部路由，或者 ASBR 配置了重分发但当前没有可引入的路由
2. **有 LSA5 必有 LSA4**：如果 ASBR 产生了 LSA5，且 ASBR 与接收 LSA5 的路由器不在同一区域，ABR 一定会产生 LSA4 来告知如何到达 ASBR
3. **无 LSA4 无 LSA5**：没有 ASBR，或者 ASBR 与接收 LSA5 的路由器在同一区域（此时不需要 LSA4）
## 2. 有 LSA5不一定有 LSA4
只有当 ASBR（自治系统边界路由器）与接收 LSA5 的路由器**不在同一个区域**时，ABR 才会产生 LSA4 来通告 ASBR 的位置信息。<font color="#7030a0">比如 AR3和 AR4接收 55.55.55.55就不会有 LSA4</font>
	 所以<font color="#ff0000">只要在同一个区域，就不会产生 LSA4</font>

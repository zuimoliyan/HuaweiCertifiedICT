
# 一、特殊区域
## 1、区域的类型
1. 骨干区域：区域0（`Area 0`）
2. 非骨干区域
	1. 常规区域（可以引入外部路由生成 `LSA5`）
	2. 特殊区域--为了减少 LSDB 的规模，达到减少路由表的规模

## 2、特殊区域的详解
### 1.stub：末节区域（<font color="#ff0000">不会接收引入的路由也不能引入路由</font>）
#### 配置命令
在指定的区域配置 `stub`
`[Huawei-ospf-1-area-0.0.0.1]stub`
#### 优点：
LSA5不能在 stub 区域进行泛洪，LSA4也不能泛洪；当外部路由震荡的时候，不会影响到 stub 区域的 PRC （PRC算法能够快速计算出最优路径，从而实现快速路由收敛）的计算。协商的时候，option 字段中，<font color="#ff0000">E=0</font>，表示 `LSA5` 无法泛洪进去。
	![](DICM/Pasted%20image%2020251210181624.png)
#### 缺点 :
访问外部路由使用默认（stub 区域，ABR 会产生一条 LSA3的缺省，产生的 LSA缺省默认开销是1，可以通过 `default cost` 修改），可能会产生次优路径，<font color="#7030a0">虽然可以改开销，但牵一发动全身</font>，所有路径都可能会变动。没办法解决这个缺点。
	如图：按照开销，AR5会走上面的路径访问 AR2，但是实际上走下面的开销会比走上面小，造成了<font color="#ff0000">次优路径</font>
	![](DICM/Pasted%20image%2020251210182917.png)
	![](DICM/Pasted%20image%2020251210184115.png)

### 2. Totally stub---完全 Stub 区
#### 配置命令
在指定区域的 `ABR` 路由器上配置 `stub no-summary`
`[Huawei-ospf-1-area-0.0.0.1]stub no-summary`

#### 优点：
LSA5不能在 stub 区域进行泛洪，LSA4和 LSA3也不能泛洪；当外部路由和区域间路由震荡的时候，不会影响到 stub 区域的 PRC （PRC 算法能够快速计算出最优路径，从而实现快速路由收敛）的计算
#### 缺点：
访问外部网络和区域间网络都使用默认，访问外部网络和区域间网络的时候都有可能出现次优路径
如图所示，LSDB 数据库变小，LSA3传输过来的路由也被过滤
	![](DICM/Pasted%20image%2020251210184334.png)


### 3. nssa 区域---非完全的末节区域
 
1. 首先，该区域有性能较差的设备，那么部署为特殊区域，但同时又需要<font color="#ff0000">外部路由</font>的时候，将这个区域部署为 `NSSA` 区域 
	**NSSA 区域内的 ASBR 将引入的外部路由，封装为 LSA7格式进行通告**。
2. `NSSA` 区域的 `ABR` 同时也是 `ASBR`
#### 配置命令
接口下配置： `[Huawei-ospf-1-area-0.0.0.1]nssa`

#### 1. Options 中N 和 E 的组合方式

|   N=0、E=0    |       N=0、E=1        |  N=1、E=0  | N=1、E=1 |
| :----------: | :------------------: | :-------: | :-----: |
|     stub     |      骨干/Normal       |   NSSA    |   不存在   |
| 不允许 `LSA4/5` | 允许 `LSA5`，不允许 `LSA7` | 允许 `LSA7` |         |

#### 2. LSA7转 LSA5（LSA7仅仅能在 NSSA 区域泛洪，不能去到其他区域）
ABR负责将 `LSA7` 转换为 `LSA5`
##### <font color="#ff0000">如果存在多个 ABR，谁做 LSA 7转 LSA5？</font>
默认情况下， `Router-ID` 大的做 `LSA7` 转 `LSA5`

#### 3. FA--转发地址

##### 开销计算
![](DICM/Pasted%20image%2020251211172559.png)
<font color="#ff0000">type = 1</font>,此时 AR2想访问 AR5引入的外部路由，计算外部路由的时候 ：
1. FA 不等于0：<font color="#ff0000">开销=外部路由开销+到 FA 的 OSPF 路由的开销 </font>
	- 外部开销是 `1`
	- AR2到 FA 的 OSPF 路由器（AR5）开销是 ：`AR2-->AR3-->AR5` `1000+1 = 10001`
	所以总开销是 `1+1001 = 1002`

2. FA 等于0：开销=<font color="#ff0000">外部路由开销+到 ASBR 距离开销</font>
**强制将 Type-7 LSA 的 FA 地址设置为 0.0.0.0**：`[Huawei-ospf-1-area-0.0.0.1]nssa suppress-forwarding-address`
	外部开销是 `1`，AR2到 AR3的开销是 `1`，所以是 `1+1 = 2`

##### NSSA 产生 FA 的条件
1. 满足三个条件：
	- 去往外部路由的接口宣告进了 OSPF 
	- 该接口没有被静默
	- 该接口不是 P2P 和 P2MP 的网络类型
2. 如果不满足
	- 首先使用环回口（一定是发布到了 OSPF）作为 FA
	- 使用物理接口地址（选择 network 物理接口 Ip 中第一个配置上去）

#### 4.NSSA 的优点（特点）
1. 防止次优路径
2. 防止出现环路

### 4. Totally NSSA
LSA5和 LSA3以及 LSA4都不会泛洪到该区域，减少了 LSDB 的规模，路由表的规模相应变小。
访问其他区域的网络信息和外部路由器信息使用缺省的 LSA3

#### 配置命令
接口下配置：`[Huawei-ospf-1-area-0.0.0.1]nssa no-summary`

## 3、案例配置
#### NSSA 配置
因为默认不出意外的情况下，NSSA 会自动执行 LSA7转 LSA5，所以我们探讨 FA 相关的问题
![](DICM/Pasted%20image%2020251210100035%201.png)
还是这个实验，其他部分与 LSA3的引入不变，<font color="#ff0000">在 LSA5</font>：
```
#
interface GigabitEthernet0/0/1
 ip address 10.1.35.5 255.255.255.0 
#
interface GigabitEthernet0/0/2
 ip address 10.1.45.5 255.255.255.0 
#
ospf 1 router-id 5.5.5.5 
 import-route static cost 100 type 1
 area 0.0.0.1 
  network 10.1.35.0 0.0.0.255 
  network 10.1.45.0 0.0.0.255 
#
ip route-static 8.8.8.8 255.255.255.255 NULL 0
```
![](DICM/Pasted%20image%2020251211163755.png)
#### 开销计算
##### 第一步：计算 “AR1 到 FA 地址（10.1.35.5）的内部开销”
`AR1→ AR2→ AR3→ AR5`
各段开销（GE 口默认开销 = 1）：
- AR1-->AR2：1
- AR2-->AR3：1
- AR3-->AR5：1
    **内部总开销 = 1+1+1 = 3**
##### 第二步：加上 “LSA7 中的外部路由开销”
从 `display ospf lsdb nssa` 的输出可知，LSA7 中携带的外部开销是 `TOS 0 Metric: 100`（即 AR5 引入静态路由时配置的 `cost 100`）。
- 如果 metric-type = 2,计算开销的时候就仅计算外部开销，<font color="#ff0000">不计算内部开销</font>
- 若为 metric-type = 1 外部路由，总开销会是 “内部开销 + 外部路径的所有开销”。
##### 最终 AR1 到 8.8.8.8 的总开销
总开销 = 内部开销 + 外部开销 = **3 + 100 = 103**

<font color="#7030a0">如果 type 为2，那么开销就只为外部开销 = 100</font>
#### 解析1：对于 NSSA 和 FA 的一般理解
此时，输入 `[Huawei]display ospf lsdb nssa` 可以发现，Options 字段被置为了 `NP`
![](DICM/Pasted%20image%2020251211163755.png)
- NP 字段是 Type - 7 LSA 向 ABR 发出的 “转换申请凭证”，N 位证明自身符合 NSSA 区域的转换前提，P 位主动请求执行转换操作。只有这两个条件同时满足，ABR 才会启动 LSA7 到 LSA5 的转换流程
- 为什么 FA 地址是 `10.1.35.5`？
	ASBR（AR5）生成 LSA7 时，FA 地址的选择规则是：优先选择**引入外部路由的接口所在的 OSPF 宣告网段的地址**，如图，`35` 被优先宣告，所以选它，如果此时关掉 `35`，那么就会选择`45`
		![](DICM/Pasted%20image%2020251211164457.png)

**FA 地址：**
1. 特点：永不为0，默认路由除外；
2. 置位条件：
	1. 先看是否满足发送 LSA7 的接口与访问外部路由的接口为同一接口的条件；
	2. 不满足上述条件，则进行自动选举：
		- 在宣告进 ospf 的 loopback 接口中选择地址最大的作为 FA 地址；
		- 在宣告进 ospf 的物理接口中选择地址最大的作为 FA 地址；
作用：防止只有一台 ABR 进行 LSA7转 LSA5时带来的次优问题；
#### 解析2：NSSA 的 FA 如何防环
LSA7 是 NSSA 区域内的 ASBR 生成的外部路由信息，它的防环关键在于 **“不能跨 NSSA 区域传播”** 和 **“P-bit 标记的限制”**

## 二、链路聚合 LAG
### 1.基本概念
1. 聚合组LAG：华为的LAG接口是`Eth-trunk`。他是一个逻辑口，包含若干个物理口。如果从逻辑接口的某个成员端口收到BUM帧，**不会泛洪给其他的成员端口**
2. 聚合接口：属于聚合组的物理接口
3. 活跃端口active（select）：可以转发数据的属于LAG逻辑端口的成员端口
4. 不活跃端口inactive（unselect）：不可以转发数据的，属于LAG逻辑端口的成员端口

### 2.链路聚合的好处
1. 增加带宽
2. 链路是冗余的，稳定性高-可以避免单点故障
3. 可以实现负载分担
	1. 基于包：所有报文轮询在成员接口上发送
		缺点是：数据流不能按顺序到达对端，那么会产生抖动
	2. 基于流：通过哈希运算来实现负载分担
		缺点是：如果流比较单一，可能会导致链路聚合组的成员端口闲置
### 3.聚合模式
#### 1.手工配置
1. 手工配置特点
	1. 优点：不需要配置协议
	2. 相同的速率
	3. 相同的双工模式
	4. ==捆绑的物理接口对端一定是同一台设备==
	5. 支持不同速率的捆绑
		- 可以设置每个接口的权重，接口带宽大配置的权重高
2. 配置命令
```
interface Eth-Trunk1
 trunkport g 0/0/1
 trunkport g 0/0/2
 ```
或者

interface Eth-Trunk1
 int g 0/0/1 
 Eth-Trunk1
 int g 0/0/2
 Eth-Trunk1
```
3. 缺点
	1. 不能确定LAG口的所有成员端口对端是同一台设备
		![](DICM/Pasted%20image%2020251114170750.png)
	2. 不能判断对端接口是否故障
#### 2.静态LACP
1. LACP-链路聚合组控制协议，封装在L2之上，使用组播报文，目的MAC`0180C2000002`
	该报文周期性发送，通过这个报文可以判断对端端口是否故障，还可以确定是否同一台交换机
2. LACP的内容
	1. LACP的优先级
	2. MAC地址
	3. 接口的优先级
	4. 接口的编号
##### 1.选举规则
1. 确定 “决策端”（谁主导选举）
2. LACP 协议中，两端设备先通过比较**系统优先级**(system priority)和**系统 ID**（MAC 地址）确定 “决策端”（Actor），由决策端主导活动端口的选择：
	1. 系统优先级数值越小，优先级越高（默认 32768，可手动配置`[Huawei]lacp priority 10`）。
	2. 若系统优先级相同，则比较系统 ID（MAC 地址越小，优先级越高）。
	3. 优先级更高的设备成为 “决策端”，另一方为 “被动端”（Partner）。
3. 决策端筛选对端 PortKey 相同的端口（同一聚合组）
4. **按 “端口优先级（小→大）→端口号（小→大）” 排序**
5. 按 “最大活动端口数” 保留前 N 个端口，其余标记为未选中
	 Eth-Trunk 可配置 “最大活动端口数”（Max Active-linknumber，默认 8），即使符合上述条件的端口数量超过该值，也只会保留优先级最高的 N 个（N 为配置值），其余标记为 “Unselect”。

##### 2.举例
1. 首选，根据`system priority`选择决策端
	- LSW1是10，LSW2是100，LSW3是50，**所以选择LSW1**
2. 再根据接口优先级选择聚合链路
	- 端口1和2的优先级是默认（32768）
	- 端口3的优先级是100
	所以选择端口3所在当聚合链路
